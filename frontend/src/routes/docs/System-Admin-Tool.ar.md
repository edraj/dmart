### **أداة إدارة النظام الرسومية (GUI)**

---

**مقدمة**

أداة إدارة النظام الرسومية (GUI) هي واجهة إدارة قوية مصممة لتسهيل إدارة الإدخالات داخل النظام. إنها توفر لمسؤولي النظام القدرة على عرض وتحديث وحذف الإدخالات بناءً على دورهم والأذونات المرتبطة بهم. تعمل هذه الأداة كمنصة مركزية للوصول إلى البيانات والتلاعب بها، على غرار التفاعل مع قاعدة البيانات. لدى DMART واجهة إدارة شاملة تتفاعل مع الخلفية بالكامل عبر واجهة برمجة تطبيقات رسمية. وهي مبنية باستخدام Svelte و Routify3 و SvelteStrap.

```bash
cd dmart/frontend
yarn install

# تكوين عنوان URL الخلفي لخادم DMART في src/config.ts

# للتشغيل في وضع التطوير
yarn dev

# للبناء والتشغيل في وضع الإنتاج / وضع تقديم الملفات الثابتة (أي بدون nodejs) باستخدام Caddy
yarn build
caddy run
```

**بناء ثنائي Tauri (تطبيق Linux AppImage)**

يتيح ذلك تجميع أداة الإدارة كتطبيق سطح مكتب.

```
# بناء عادي بدون فحص
yarn tauri build --bundles appimage

# لإضافة الفحص (انقر بزر الماوس الأيمن -> فحص)
yarn tauri build --bundles appimage --debug

```

**الميزات الرئيسية**

1. **إدارة الإدخالات**: يمكن لمسؤولي النظام إنشاء وعرض وتحديث وحذف الإدخالات داخل النظام. يشمل هذا القدرة على إدارة أنواع مختلفة من الإدخالات مثل المحتوى والتذاكر والمرفقات.

2. **تعريف المخطط**: تتيح الأداة للمستخدمين تعريف المخططات، والتي تعمل كقوالب لتنظيم البيانات وبنائها. يمكن للمسؤولين إنشاء مخططات مخصصة لتناسب المتطلبات الخاصة لمؤسستهم.

3. **إعداد سير العمل**: يمكن للمسؤولين تعريف سير العمل داخل النظام، وتحديد الخطوات والعمليات المطلوبة لإكمال المهام أو حل المشكلات. بعد ذلك، يمكن للمستخدمين إنشاء تذاكر مرتبطة بهذه سير العمل.

4. **إدارة الأدوار والأذونات**: توفر الأداة وظيفة لتحديد أدوار وأذونات جديدة داخل النظام. يمكن للمسؤولين تعيين أدوار للمستخدمين والتحكم في مستويات وصولهم وإدارة الأذونات بناءً على متطلبات المؤسسة.

5. **إدارة المستخدمين**: باستخدام مساحة الإدارة، يمكن للمسؤولين إنشاء وتنشيط وإلغاء تنشيط وتحديث وحذف حسابات المستخدمين. يشمل هذا إدارة معلومات المستخدم والأدوار والأذونات.

6. **إعداد التطبيق**: يمكن للمسؤولين إدارة إعدادات وتكوينات تطبيقات مختلفة داخل النظام. يشمل هذا تحديث السجلات والترجمات والتكوينات والإعلانات والمعلمات الأخرى المتعلقة بالنظام.

1. **إنشاء دور**: حدد الأدوار الجديدة داخل النظام من خلال تحديد اسمها ووصفها والأذونات المرتبطة بها.
2. **تعيين الدور**: قم بتعيين الأدوار للمستخدمين عن طريق تحديد المستخدم وتعيين الدور (الأدوار) المطلوب من الخيارات المتاحة.
3. **إدارة الأذونات**: التحكم في مستويات وصول المستخدم والأذونات عن طريق تعديل تعيينات الأدوار وإعدادات الأذونات.

**إدارةالمستخدم**

1. **إنشاء مستخدم**: أضف مستخدمين جدد إلى النظام من خلال توفير التفاصيل الخاصة بهم، بما في ذلك اسم المستخدم وكلمة المرور والبريد الإلكتروني وتعيينات الأدوار.
2. **تنشيط/إلغاء تنشيط المستخدم**: تمكين حسابات المستخدمين أو تعطيلها بناءً على المتطلبات التنظيمية.
3. **تحديث معلومات المستخدم**: تعديل تفاصيل المستخدم، بما في ذلك معلومات الاتصال والأدوار والأذونات.
4. **حذف المستخدم**: إزالة حسابات المستخدمين من النظام، ونقل بياناتهم أو أرشفتها اختياريًا حسب الضرورة.

** تكوين التطبيق **

1. **تحديث السجلات**: إدارة سجلات النظام لتتبع ومراقبة نشاط النظام والأخطاء ومقاييس الأداء.
2. **إدارة الترجمات**: تحديث ترجمات اللغة لواجهات المستخدم ورسائل النظام.
3. ** تكوين الإعدادات **: تعديل تكوينات النظام والمعلمات لتخصيص سلوك التطبيق ومظهره.
4. **نشر الإعلانات**: قم بإنشاء ونشر الإعلانات لتوصيل المعلومات أو التحديثات المهمة لمستخدمي النظام.

**خاتمة**

توفر أداة إدارة النظام (GUI) لمسؤولي النظام مجموعة شاملة من الأدوات لإدارة الإدخالات والمخططات وسير العمل والأدوار والأذونات والمستخدمين وتكوينات التطبيق داخل النظام. ومن خلال الاستفادة من هذه الواجهة البديهية، يمكن للمسؤولين تنظيم بيانات النظام وإعداداته والتحكم فيها وصيانتها بكفاءة لتلبية احتياجات مؤسستهم.
